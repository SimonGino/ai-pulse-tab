## Why

当前页面存在四个核心问题：1) 左侧用量卡片内容密集、右侧书签区域大量留白，布局严重失衡；2) 同时使用黄、橙、青、绿、红、粉等高饱和霓虹色，视觉杂乱刺眼；3) 像素字体 "Press Start 2P" 应用于数据密集区域（用量百分比、计划名称等），可读性差；4) 缺少点击跳转、信息层级区分等基础交互体验。作为用户每天打开几十次的新标签页，这些问题严重影响日常使用效率。

## What Changes

- **布局重构**：将书签栏移至搜索栏下方，做成全宽一行；用量卡片改为全宽布局，左右并排 Claude 和 ChatGPT；去掉或大幅缩小页面标题
- **配色精简**：主色控制在 2-3 个（青色为主色、橙色为警告色、绿色为正常态），降低纯色饱和度（`#00ff00` → `#4ade80`），RESET 倒计时改用柔和灰色
- **字体策略**：标题和装饰保留像素字体，数据区域（用量百分比、计划名称、倒计时等）改用等宽可读字体（如 JetBrains Mono / Fira Code）
- **进度条优化**：加宽进度条，增强视觉辨识度；卡片内各用量指标之间增加视觉分隔
- **交互增强**：点击 Provider 卡片标题可跳转到对应服务；"Last updated" 改为不显眼的 tooltip 或缩小展示

## Capabilities

### New Capabilities

（无新增能力，全部为现有能力的优化）

### Modified Capabilities

- `dashboard-layout`：布局结构重构——书签移至搜索栏下方全宽展示，用量卡片改为全宽左右并排，标题缩小或移除
- `pixel-theme`：配色精简为 2-3 主色，数据区域改用可读等宽字体，进度条加宽，卡片内信息层级增强
- `quick-links`：书签栏位置从右侧栏改为搜索栏下方全宽一行展示

## Impact

- **组件改动**：`App.tsx`（布局结构）、`ProviderCard.tsx`（点击跳转、信息层级）、`QuotaBar.tsx`（进度条样式）、`BookmarkGrid.tsx`（位置调整）、`SearchBar.tsx`（与书签栏的间距协调）
- **样式改动**：`style.css`（CSS 变量、字体声明、布局网格）、各组件内联样式
- **依赖变更**：可能需要引入 Google Fonts 中的 JetBrains Mono 或 Fira Code
- **无 API 或数据结构变更**
