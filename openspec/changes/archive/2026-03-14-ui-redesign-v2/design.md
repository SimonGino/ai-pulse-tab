## Context

AI Pulse Tab 是一个浏览器新标签页扩展，使用 WXT + React + Tailwind CSS 构建，采用复古像素风格（"Press Start 2P" 字体 + 霓虹配色）。当前版本存在布局失衡（左重右轻）、配色过杂（6+ 种高饱和霓虹色）、数据区像素字体可读性差、缺少基础交互（点击跳转）等问题。

技术栈：WXT 0.20、React 19、Tailwind CSS 4、TypeScript。样式通过 CSS 变量（`style.css`）+ Tailwind 工具类 + 组件内联样式混合实现。

## Goals / Non-Goals

**Goals:**
- 重构布局为"搜索栏 → 书签栏（全宽）→ 用量卡片（全宽并排）"的垂直流
- 将配色精简为 3 个主色（青色主色、绿色正常态、橙色/红色警告态），降低饱和度
- 数据区域改用可读等宽字体，标题和装饰保留像素字体
- 加宽进度条、增加卡片内信息分隔、添加 Provider 点击跳转

**Non-Goals:**
- 不改变数据获取逻辑（probe、background worker、storage schema 不变）
- 不新增功能模块（如书签分组、搜索历史等）
- 不更换整体主题风格（仍为暗色复古风，只做精细化调整）
- 不改变移动端/窄屏的单列回退行为

## Decisions

### 决策 1：布局结构 — 垂直流 vs 保持双列

**选择：垂直流布局**

将书签从右侧栏移至搜索栏下方全宽展示，用量卡片做成全宽区域内左右并排（桌面端两列 grid）。

- **理由**：当前右侧书签只有一行图标，大量留白；而左侧用量卡片内容密集。垂直流让每个区域都能充分利用全宽，信息密度更均匀
- **替代方案**：保持双列但调整比例（7:3）——但书签内容太少，即使压缩右栏仍会有大量留白
- **影响**：`App.tsx` 的 grid 结构从 `grid-cols-2` 改为垂直排列；`BookmarkGrid` 组件位置移动；CSS `.dashboard-grid` 类重写

### 决策 2：字体策略 — 双字体 vs 全部替换

**选择：双字体方案**

引入 JetBrains Mono 作为数据字体（通过 Google Fonts CDN），保留 "Press Start 2P" 用于标题和装饰元素。

- **理由**：完全去掉像素字体会丢失主题辨识度；仅在可读性关键区域（百分比数值、计划名称、倒计时）使用等宽可读字体
- **替代方案**：使用系统等宽字体（`ui-monospace, monospace`）——无需外部依赖，但视觉一致性差
- **影响**：`style.css` 新增 `--font-data` 变量；需要添加新的 CSS 类 `.data-font`；`QuotaBar`、`ResetCountdown`、`ProviderCard` 的数据文本应用新字体类

### 决策 3：配色方案 — 精简策略

**选择：3 主色 + 2 辅助色**

| 角色 | 现有 | 调整后 |
|------|------|--------|
| 正常/低用量 | `#00ff00` | `#4ade80`（Tailwind green-400） |
| 中用量/警告 | `#ffb852` | `#fb923c`（Tailwind orange-400） |
| 高用量/危险 | `#ff0000` | `#f87171`（Tailwind red-400） |
| 主强调色 | `#00ffff` | `#22d3ee`（Tailwind cyan-400） |
| 文字/次要 | `#e0e0e0` / `#555555` | 保持不变 |
| 倒计时文字 | `#ff0000`（红色） | `#9ca3af`（Tailwind gray-400） |

- **理由**：降低饱和度减少视觉疲劳；使用 Tailwind 400 色阶在深色背景上可读性好且不刺眼；倒计时不是告警信息，不应用红色
- **移除**：`--pixel-yellow`（`#ffff00`）和 `--pixel-pink`（`#ffb8ff`）从主要 UI 中移除，仅保留在 Pacman 装饰中

### 决策 4：进度条改进

**选择：加宽条状 + 保持分段设计**

将 QuotaBar 从 10 个小方块改为更宽的条状（高度从 ~8px 提升到 ~16px），保持 10 段分段但视觉上更连续。段间间距缩小。

- **理由**：当前方块太小在高分屏上辨识度低；加宽后百分比一目了然
- **影响**：`QuotaBar.tsx` 样式调整

### 决策 5：页面标题处理

**选择：大幅缩小标题**

将 "AI Pulse Tab" 缩小为页面左上角的小标签，不再居中大字展示。

- **理由**：用户每天看几十次，标题信息价值低；但完全去除会让页面缺少品牌标识
- **影响**：`App.tsx` 标题区域重构

## Risks / Trade-offs

- **[字体加载延迟]** → 使用 `font-display: swap` 确保字体加载前先用系统等宽字体显示；字体文件较小（JetBrains Mono 仅加载 latin 子集）
- **[布局变动较大]** → 窄屏下的单列布局基本不受影响，变动主要在桌面端宽屏；通过 CSS media query 隔离
- **[用量卡片并排可能内容不等高]** → 使用 CSS Grid 的 `align-items: start` 避免强制等高，各卡片自然高度即可
- **[现有用户习惯]** → 变动是视觉层面的，功能和数据不变，用户适应成本低
