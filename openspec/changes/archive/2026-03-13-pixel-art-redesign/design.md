## Context

AI Pulse Tab 是一个浏览器扩展的新标签页，当前使用 React + Tailwind CSS 构建，界面为暗色主题的简洁卡片布局。页面仅包含 Claude 用量展示（ProviderCard + QuotaBar + ResetCountdown），视觉表现较为单调。

用户希望将整个页面改造为吃豆人风格的像素艺术主题，同时增加常用网站快捷跳转功能，提升新标签页的趣味性和实用性。

## Goals / Non-Goals

**Goals:**
- 全局 UI 转换为 8-bit 像素风格，包括字体、边框、按钮、进度条
- 添加吃豆人主题装饰元素（角色、豆子、迷宫线条等）
- 新增快捷链接区域，支持预设常用网站跳转
- 保持现有用量数据展示功能不受影响

**Non-Goals:**
- 不做完整的吃豆人游戏（仅做装饰性动画）
- 不支持用户自定义链接（首版仅预设）
- 不更改数据获取逻辑和后端 probe 代码
- 不做主题切换功能（首版仅像素主题）

## Decisions

### 1. 像素字体方案：Google Fonts "Press Start 2P"

**选择**: 使用 Google Fonts 提供的 "Press Start 2P" 像素字体
**替代方案**:
- 自托管像素字体文件 → 增加扩展体积
- 纯 CSS 模拟像素效果 → 效果不佳，难以实现真实像素感
**理由**: Press Start 2P 是最经典的像素字体之一，Google Fonts CDN 加载快且免费，支持 ASCII 和部分 Unicode 字符。中文内容继续使用系统默认字体。

### 2. 像素化 UI 方案：CSS + Tailwind 自定义样式

**选择**: 通过 Tailwind CSS 自定义类实现像素风格（像素边框用 box-shadow 模拟、方形圆角、8-bit 配色）
**替代方案**:
- 使用 NES.css 等像素 UI 框架 → 引入额外依赖，可能与现有 Tailwind 冲突
- 全部使用图片切图 → 不灵活，维护成本高
**理由**: 项目已使用 Tailwind CSS，通过自定义样式可保持技术栈统一，无需额外依赖。

### 3. 吃豆人装饰：CSS 动画 + 纯 CSS 像素绘制

**选择**: 使用纯 CSS 绘制吃豆人角色和豆子，CSS animation 实现移动动画
**替代方案**:
- Canvas 绘制 → 过于复杂，维护难度大
- SVG 动画 → 不够像素风
- GIF 图片 → 不灵活，文件体积大
**理由**: 纯 CSS 方案轻量无依赖，且能实现足够逼真的像素效果。吃豆人和幽灵用 CSS box-shadow 逐像素绘制。

### 4. 快捷链接布局：网格卡片式

**选择**: 在用量卡片下方添加网格布局的快捷链接区域，每个链接为一个像素风格卡片
**理由**: 网格布局直观清晰，与上方的用量卡片视觉一致，且适合展示多个链接。

### 5. 快捷链接图标：CSS 像素图标

**选择**: 用 CSS box-shadow 技术绘制各网站的简易像素图标，或使用首字母像素化展示
**替代方案**:
- 使用网站 favicon → 风格不统一，破坏像素美感
- 使用第三方图标库 → 无像素风格图标
**理由**: 保持全局像素风格一致性，自制像素图标更有趣味性。

## Risks / Trade-offs

- **[像素字体可读性]** → Press Start 2P 字号较小时可读性差，数据展示部分适当增大字号，中文内容使用系统字体
- **[Google Fonts 加载延迟]** → 使用 font-display: swap 避免 FOIT，首次加载时会闪烁一次
- **[CSS 像素绘制性能]** → box-shadow 像素绘制大量元素时可能影响性能，控制装饰元素数量和复杂度
- **[扩展 CSP 限制]** → 浏览器扩展可能限制外部字体加载，需在 manifest 中配置 CSP 允许 Google Fonts 域名，或改用内联字体
